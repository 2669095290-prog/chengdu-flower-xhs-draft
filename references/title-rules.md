# Title Rules

## Hard Rules

- End every title with `成都花店` unless the user overrides it.
- Include one emoji before `成都花店` or near the emotional hook.
- Aim for the platform title limit: about 20 visible characters. Use `node -e 'console.log([...title].length,title.length,title)'` to check.
- Prefer user-view titles over merchant-view titles.
- Avoid bland directory titles like `成都花店｜粉色花束适合送礼`.

## Better Hooks For Flower Posts

Use phrases that feel like users saw or received the bouquet:

- `这么漂亮的花到底是谁收到了呀💗成都花店`
- `收到这束粉红花真的会很开心💗成都花店`
- `收藏夹里的花被送到了🎀成都花店`
- `直男送花终于开窍了💐成都花店`
- `这束真的被温柔到了💗成都花店`
- `收到这束抱着不想撒手💗成都花店`

## Formula Bank

- Received feeling: `收到这束{颜色/风格}花真的{情绪}💗成都花店`
- Curiosity: `这么漂亮的花到底是谁收到了呀💗成都花店`
- Boyfriend angle: `男朋友送这束花算会了💐成都花店`
- Scene angle: `见女友前顺手买了束花💐成都花店`
- Contrast angle: `别再只会送红玫瑰了🩵成都花店`
- Mark/collection angle: `漂亮花束先Mark住💐成都花店`

## Image-To-Title Mapping

- 奶油粉白/温柔玫瑰: use `温柔到`, `心软`, `奶油粉白`, `乖`.
- 粉红/玫红/甜感花束: use `谁收到了呀`, `少女心`, `小惊喜`, `会开心`.
- 蓝白/蝴蝶兰/高级冷调: use `高级到失语`, `别送红玫瑰`, `清醒浪漫`.
- 多图合集: use `Mark一些`, `3束`, `不撞款`, `不翻车`, only if the images support it.

## Self-Check

Before using a title, ask:

- Does it sound like a real Xiaohongshu note, not a merchant catalog?
- Does it include a human scene or emotional reaction?
- Does it still preserve the local search term `成都花店`?
- Is it close to 20 visible characters without feeling padded?
